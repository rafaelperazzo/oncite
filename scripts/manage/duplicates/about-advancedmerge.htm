Using the advanced merge tool

The data in the drop-down menus represent areas of differences between the duplicated records. In many cases the difference may only be a simple spelling mistake, or an abbreviation used in one record versus the full text in another. With authors, the difference may only be that a comma or period was omitted from one record.

Where there is no drop-down menu then the data is the same in all of the selected records (blank fields are ignored).

Advice for specific fields:

  • Originator: The user IDs of the records' originators are stored, but only one can be kept in the merged record. Choose your own user ID if it appears there - otherwise choose whatever seems most appropriate.
  • Pubid: This lists the ids of the duplicated records. The merged record will take on the id that you select. The publication id that is not selected will be deleted.
  • URL: If there are multiple URLs then you may need to exercise judgment about which is the best to keep. PubMed URLs (shown as "pm:" followed by a number) are often useful to keep since they may link to full-text options. (Eprints records will also have extra fields - "oaiid" and "eprint" - which are used to create additional Eprints-derived links. These are separate from the URL field.)

After you have made any selections and elected to merge the records, you will still be able to edit the record to make any further adjustments.
